# PRD — Product Requirements Document

## Executive Summary

Restoloop is a WhatsApp-first customer outreach SaaS for Indian restaurants. , fast, mobile-friendly owner dashboard. The backend (Supabase, cron campaigns, WhatsApp API) remains untouched.

## Problem Statement

Restaurant owners need a simple way to:
1. Collect customer data (name, phone, birthday)
2. Auto-send WhatsApp coupons (welcome, birthday, winback)
3. Track who received what, without learning complex software



## Goals & Success Metrics

| Goal | Metric |
|------|--------|
| Owner understands dashboard in 10 seconds | Time-to-first-action < 10s |
| Zero onboarding friction | No setup wizard, immediate use after sign-in |
| Mobile-friendly | All owner pages usable on 375px width |
| Intake form converts visitors | Form completion rate > 60% |
| No backend changes | 100% frontend-only delivery |

## User Personas

**Primary: Restaurant Owner (Raju, 45)**
- Runs a single-location restaurant in India
- Uses smartphone + POS for everything
- Not tech-savvy, wants "set and forget"
- Needs to see customers, coupons, credits at a glance

**Secondary: Customer (end user)**
- Dines at restaurant, scans QR or visits link
- Fills intake form to get welcome coupon
- Receives WhatsApp with coupon code

## Product Requirements

### Auth
- Email + password sign-up/sign-in
- Password reset via email
- No OAuth, no captcha for MVP
- Plain HTML forms, no fancy containers

### Dashboard (`/home`)
- 3 KPI cards: Total Customers, Coupons Sent This Month, Credits Remaining
- Activity feed: recent sends, birthdays today, low credit alert
- Optional: line chart for message sends over last 30 days

### Customers (`/home/customers`)
- Table: Name, Phone, Birthday, Last Visit, Visit Count, Customer Since
- Sortable columns
- Search/filter by name/phone
- No pagination for MVP (< 500 customers typical)
- View only, no inline editing

### Coupons (`/home/coupons`)
- Read-only list: Code, Type, Discount, Status, Sent Date, Customer
- Filter by type (welcome / birthday / winback)
- No manual creation — all auto-generated by cron

### Profile (`/home/profile`)
- Restaurant name, address, phone, email
- Coupon values (discount amounts)
- Editable, saves to Supabase

### Public Intake Form (`/form/[slug]`)
- Unauthenticated, per-tenant via slug
- Fields: Name, Phone, Birthday (optional), Favorite Dish (optional)
- Auto-generates welcome coupon
- Sends WhatsApp message with coupon code
- Success state with coupon display

## Non-Functional Requirements

- **Performance**: Page load < 2s on 3G
- **Accessibility**: WCAG 2.1 AA minimum
- **Mobile-first**: All layouts designed 375px up
- **Browser support**: Last 2 Chrome, Safari, Firefox
- **No PWA requirements** for MVP

## Out of Scope

- OAuth providers
- SMS / Email channels
- Advanced analytics / charts
- Mobile native app
- Multi-location per tenant
- Staff / team accounts
- Advanced coupon rules (usage limits, stacking)
- Subscription billing (pay-per-credit only)
- POS billing module
- i18n

## Dependencies & Risks

| Dependency | Risk | Mitigation |
|------------|------|------------|
| Supabase auth SSR | Breaking changes in `@supabase/ssr` | Pin version, test auth flow |
| WhatsApp API rate limits | Campaign delays | Retry logic exists in backend |
| Meta Cloud API policy changes | Message delivery failure | 3rd-party fallback already built |
| Owner mobile data connectivity | Slow loads | Aggressive caching, minimal JS |

## Post-Launch Plan

1. **Week 1**: Monitor intake form conversion rate, fix friction points
2. **Week 2**: Add QR code share + copy link on intake form
3. **Week 3**: Build line chart for message sends (if not in MVP)
4. **Month 2**: Owner feedback survey, prioritize next feature
